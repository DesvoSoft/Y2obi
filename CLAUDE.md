# Y2obi — Project Handover

## Status: v3 — Live on Render (https://y2obi.onrender.com) + Desktop

## Stack
- **Desktop**: CustomTkinter 5.2.2, Python 3.14
- **Web**: Flask 3 + gunicorn 2 workers, Vitra CSS v1.7.1 (glassmorphism, particles, themes)
- **Engine**: yt-dlp v2026.6.9
- **Mux/Convert**: FFmpeg (auto-download on Windows; system binary on Linux/Render)
- **Build**: PyInstaller -> single `.exe`
- **Deploy**: Render (free tier, Virginia US East)
- **Repo**: https://github.com/DesvoSoft/Y2obi

## Render Deploy Config
- **Build command**: `pip install -r requirements-web.txt`
- **Start command**: `gunicorn web.app:app --workers 2 --timeout 120 --bind 0.0.0.0:$PORT`
- **Env vars**:
  - `SECRET_KEY`: auto-generated by Render
  - `COOKIES_B64`: base64-encoded cookies.txt (optional but needed for bot bypass)
  - `DOWNLOAD_DIR`: `/tmp/y2obi-web` (default)
- **Filesystem**: ephemeral. Downloads go to `/tmp/y2obi-web/`, deleted after 10min.
- **Sleep**: free tier sleeps after 15min inactivity. In-progress downloads survive within same instance.

## Web Data Flow
1. `GET /` -> render Vitra SPA
2. `POST /api/analyze` -> `_make_dl().get_info(url)` (fresh Downloader reads cookies from disk)
3. UI shows: thumb, title, channel, duration, views, likes, quality list
4. `POST /api/download` -> spawn thread, return `task_id`
5. `GET /api/progress/<id>` -> poll every 500ms
6. `POST /api/cancel/<id>` -> set cancel flag
7. `GET /api/file/<id>` -> serve file, schedule delete
8. Cookie badge in footer -> upload `cookies.txt` manually

## Cookie System
- **Web upload**: `POST /api/cookies` saves to `web/cookies/cookies.txt`
- **Env var**: `COOKIES_B64` = base64 of cookies.txt, decoded at startup to `web/cookies/cookies.txt`
- **Per-request**: `_make_dl()` checks `os.path.exists(COOKIES_PATH)` fresh on each request (fixes gunicorn multi-worker issue)
- **Desktop**: checks `cookies.txt` next to `.exe` at startup
- **Chrome DPAPI**: broken on Windows. Use Firefox export or manual extension: https://chromewebstore.google.com/detail/get-cookiestxt-locally/cclelndahbckbenkjhflpdbgdldlbecc

## yt-dlp Client Strategy
- **`get_info`** (metadata only): always `android_vr` client. No format selection.
- **`download_mp4/webm`**: `web` client when cookies present, `android_vr` when not.
- **QUALITY_MAP**: tuples `(max_height, None)` + `format_sort`. No strict `ext=mp4` constraint.

## OPEN BUG (as of 2026-06-20)
`get_info` with cookies returns "Requested format is not available" on some videos.
- Confirmed: cookies reach server (`exists=True` in logs)
- Hypothesis: `android_vr` + cookies combo triggers a different format set that confuses yt-dlp's internal validation even without a format string
- Next to try: add `'format': 'bestvideo+bestaudio/best'` explicitly to `get_info` opts, OR switch to `mweb` client for get_info with cookies

## Quality Mapping
`QUALITY_MAP` = dict of `(max_height, None)` tuples. `download_mp4` uses `format_sort` + `bestvideo[height<=N]+bestaudio/best[height<=N]`. MP3: `bestaudio/best` -> FFmpeg -> 320kbps mp3.

## CSS Conventions
- `box-sizing: border-box` global reset
- `min-width: 0` on flex/grid children
- No `@import` in CSS
- No emoji in source files
- All colors via `var(--vitra-color-*)` CSS custom properties

## Build to EXE
```
pip install pyinstaller
pyinstaller build.spec
# Output: dist/Y2obi.exe
```
`core/ffmpeg.exe` NOT bundled. Auto-downloads on first run (Windows only).

## Output Directory
- Desktop: `C:\Users\<user>\Videos\Y2obi\`
- Web: `/tmp/y2obi-web/` (ephemeral, 10min cleanup)

## Known Limitations
1. No playlist support — single video only
2. No thumbnail caching
3. Chrome DPAPI broken for auto-export — use Firefox or manual export
4. Render free tier: sleeps after 15min, tasks lost on sleep if mid-download
5. gunicorn 2 workers: tasks dict in memory not shared between workers (download progress may 404 if wrong worker answers poll)

## Commands
```
pip install -r requirements.txt
python main.py           # desktop
python web/app.py        # web dev (localhost:5000)
pip install -r requirements-web.txt && gunicorn web.app:app  # web prod
pyinstaller build.spec   # build exe
```
